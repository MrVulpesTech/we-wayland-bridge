# 30.01 — Output Backends

> **Status: stub.** This document is filled in during Session 2 (the
> architecture survey). Until then it records only what is already known
> from the directory layout and the project brief. Do not treat the
> contents below as a verified map of the renderer — verify against the
> submodule source when filling this in.

## Purpose

Map how `linux-wallpaperengine` decides "where to draw", so the new
headless→PipeWire backend (ADR-0003) can be added as a sibling without
disturbing the existing targets. The deliverable of Session 2 is this
document, complete, plus a check of upstream issues/PRs for a PipeWire or
headless backend already in progress (so the project joins rather than
duplicates).

## Known from layout (unverified detail)

The renderer source groups output handling under:

```text
upstream/src/WallpaperEngine/Render/Drivers/
├── Output/        ← output targets ("where do frames go")
├── Detectors/     ← environment detection (which backend applies)
└── ...
```

The existing output targets, per the project brief, are:

- a **window** backend (`--window`), portable to any compositor;
- a **`wlr-layer-shell`** backend, for wlroots compositors (Hyprland,
  Sway), which also uses `xdg-output-unstable-v1` for monitor placement;
- an **X11** path.

GNOME (Mutter) supports none of these as a background-layer path on
Wayland, which is the reason this project exists.
