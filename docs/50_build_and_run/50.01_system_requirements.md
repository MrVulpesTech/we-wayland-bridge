# 50.01 — System Requirements

Target machine, runtime requirements, and the renderer's build
dependencies. The dependency list is finalized in `session-01-build.md`
once the build succeeds on the reference machine — until then, package
names are upstream's Ubuntu 24.04 list and are verified/adjusted for
26.04 in Session 1.

## Reference machine

| Thing | Value |
|---|---|
| OS | Ubuntu 26.04 LTS |
| GNOME Shell | 50.1 |
| Session | Wayland (Mutter), X11 session removed |
| GPU | Intel CometLake-U UHD Graphics (integrated) |
| Audio/video bus | PipeWire (present; exact version recorded in Session 1) |

The GPU vendor matters for the dma-buf transport work (ADR-0004): the
zero-copy path is proven on this integrated Intel GPU first, and that
does not generalize to NVIDIA or multi-GPU setups without further testing.

## Renderer runtime requirements (from upstream)

- OpenGL 3.3 support
- A Wallpaper Engine installation in Steam, with subscribed wallpapers
  (the renderer reads files the operator owns; it ships none)

## Build dependencies (Ubuntu, to verify on 26.04 in Session 1)

Upstream lists these for Ubuntu 24.04. Session 1 confirms the exact
package names on 26.04 and records the working `apt` line.

```text
build-essential cmake
libxrandr-dev libxinerama-dev libxcursor-dev libxi-dev libxxf86vm-dev
libgl-dev libglew-dev freeglut3-dev libglm-dev libglfw3-dev
libsdl2-dev
liblz4-dev
libavcodec-dev libavformat-dev libavutil-dev libswscale-dev
libmpv-dev mpv
libpulse-dev
libfftw3-dev
libfreetype-dev
```

Library highlights and why the renderer needs them:

| Dependency | Used for |
|---|---|
| OpenGL / GLEW / GLFW / GLM | Core rendering |
| SDL2 | Windowing / input abstraction |
| FFmpeg (`libav*`, `swscale`) | Video-type wallpapers and media decode |
| mpv (`libmpv`) | Video playback |
| LZ4, Zlib | `.pkg` archive decompression |
| FFTW3 | Audio-reactivity (FFT); not used by the MVP but a build dep |
| PulseAudio (`libpulse`) | Audio capture for reactivity |
| FreeType | Text rendering in scenes |

Xrandr/Xinerama/Xcursor are X11 libraries the renderer still links for its
X11 path; they are build dependencies even though the target session is
Wayland.

## Transport-side requirements (later sessions)

For the PipeWire output backend (ADR-0003/0004), additional development
packages will be needed — PipeWire dev headers, and on the consumer side
GStreamer with `pipewiresrc`. These are added to this list when Session 2
scopes the backend; they are not required to build the unmodified
renderer in Session 1.
