# 20.01 — Tech Stack

What each component is written in and why. The stack is dictated by the
two endpoints — a C++ OpenGL renderer on one side, the GJS-based GNOME
Shell on the other — and the IPC that joins them.

## Components

| Layer | Language / runtime | Why it is fixed |
|---|---|---|
| Renderer | C++17, OpenGL 3.3, CMake | This is `linux-wallpaperengine`. We do not choose it; we extend it. See `upstream/`. |
| Output backend (new) | C++17, in the renderer's tree | Must link the renderer's GL context and buffers; lives beside the existing output drivers (ADR-0003). |
| Frame transport | PipeWire (C API), dma-buf | The standard Wayland-era mechanism for sharing GPU buffers between processes zero-copy (ADR-0004). |
| GNOME Shell extension | GJS (GNOME JavaScript), Clutter, GStreamer | The only API surface that can place an actor in the GNOME background layer is the shell's own JS runtime. |
| Frame sink in the shell | Clutter actor fed by a PipeWire source | GStreamer's `pipewiresrc` plus a Clutter/Cogl sink, or a direct PipeWire consumer. Validated in Session 3–4. |

## Why PipeWire as the seam

PipeWire is already the system audio/video bus on a modern Ubuntu GNOME
desktop (confirmed present on the target machine). It supports dma-buf
buffer sharing, negotiates formats between producer and consumer, and has
a GStreamer element (`pipewiresrc`) on the consumer side. Using it means
neither end invents a custom shared-memory protocol, and the renderer's
new output backend is useful to consumers other than GNOME.

## Why the extension is GJS and not anything else

GNOME Shell extensions run *inside* `gnome-shell`. The runtime is GJS;
the scene graph is Clutter. There is no alternative language or process
model for code that draws in the background layer on GNOME Wayland. This
is a constraint, not a preference — see `10.01_project_vision.md`.

## Build tooling

- Renderer: CMake, system compiler (`g++`), system OpenGL/SDL2/FFmpeg/mpv
  development packages. Dependency list and versions in
  `50_build_and_run/50.01_system_requirements.md`.
- Extension: no build step required for plain GJS; packaged with
  `gnome-extensions pack` for distribution. Any TypeScript or bundling is
  deferred until the extension is more than a prototype.

## Versions on the reference machine

Recorded here so API assumptions are traceable. Verify on each machine;
the GNOME version in particular drives the extension API surface.

| Thing | Version (reference machine) |
|---|---|
| OS | Ubuntu 26.04 LTS |
| GNOME Shell | 50.1 |
| Session | Wayland (Mutter) |
| GPU | Intel CometLake-U UHD Graphics (integrated) |
| PipeWire | present (see `50.01` for exact version) |

The integrated Intel GPU matters for the zero-copy work: dma-buf sharing
and format negotiation behave differently across vendors, and "works on
Intel" does not prove "works on NVIDIA". Noted as a risk, not yet tested.
