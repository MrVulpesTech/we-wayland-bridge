# 20.03 — Component Map

The data path from wallpaper files on disk to animated pixels behind the
desktop icons, and where the process and licence boundaries fall.

## The pipeline

```text
  Wallpaper Engine files (Steam Workshop)
  project.json · scene.pkg · shared assets
              │
              │  read-only, via gitignored symlinks
              ▼
  ┌─────────────────────────────────────────────┐
  │  linux-wallpaperengine  (renderer process)   │   GPLv3
  │                                              │
  │  scene parse → HLSL→GLSL → OpenGL render     │
  │  ┌────────────────────────────────────────┐  │
  │  │  Output backend (NEW, this project)     │  │   bridge/  · GPLv3
  │  │  renders into an offscreen FBO instead   │  │
  │  │  of a window, exports the GL buffer as   │  │
  │  │  a dma-buf, publishes it to PipeWire     │  │
  │  └────────────────────────────────────────┘  │
  └─────────────────────────────────────────────┘
              │
              │  PipeWire video stream, dma-buf handle (zero-copy)
              │  ── PROCESS BOUNDARY ── ── LICENCE BOUNDARY ──
              ▼
  ┌─────────────────────────────────────────────┐
  │  gnome-shell  (extension runs inside it)     │   MIT
  │                                              │
  │  PipeWire consumer → Clutter actor in the    │
  │  background layer, beneath desktop icons      │
  │  + lifecycle: autostart, pick wallpaper,      │
  │    pause on fullscreen                         │
  └─────────────────────────────────────────────┘
              │
              ▼
  Live wallpaper on the GNOME Wayland desktop
```

## Process boundaries

There are two processes, and they stay two processes (principle P3):

| Process | Contains | Why separate |
|---|---|---|
| Renderer | `linux-wallpaperengine` + the new output backend | GPLv3; runs untrusted shader code; a crash here must not take down the desktop. |
| `gnome-shell` | The extension (a Clutter actor + lifecycle glue) | The only place that can draw in the background layer; MIT; must not link GPLv3 code. |

The PipeWire stream is the only thing crossing between them. It is both
the IPC seam and the licence seam — see ADR-0002.

## The parallel path we do not own

On wlroots compositors, `linux-wallpaperengine` already reaches the
background directly:

```text
linux-wallpaperengine ──► wlr-layer-shell ──► wlroots compositor (Hyprland/Sway)
```

That path needs nothing from this project. The output backend we add is a
*sibling* to the existing window and layer-shell backends, not a
replacement (ADR-0003). The job is to add a path for GNOME without
breaking the path that already works elsewhere.

## Where the technical risk concentrates

Everything except the **dma-buf zero-copy hop** is either already built
(the renderer) or well-trodden (a Clutter actor, a GNOME extension
lifecycle). The unproven link is exporting the renderer's GL output as a
dma-buf, publishing it through PipeWire, and importing it into Cogl/Clutter
without a CPU copy — across the specific GPU/driver on the target machine.
That is what Session 4 validates and what ADR-0004 commits to provisionally.
