## Notes

- **Q-1 is the project.** Everything else is tractable engineering; the
  zero-copy hop is the part that could prove the architecture wrong.
  Prototype it as early as the plan allows.
- **Q-9 resolved in Session 2b** (build on `next-v2` core, not `main`).
  See `30_rendering/30.02_next-v2-core-api.md`.
- **Q-10 is now the pivotal strategic question.** The technical path is
  clear; what is not is whether a separate project is the right vehicle,
  given a working window-clone solution already exists and an official
  GNOME extension is planned upstream. The #302 comment
  (`90_upstream/comment-pr609.md`) is how we get the maintainer's input
  before committing further.
- Questions move to a decision (an ADR or a session log) rather than being
  edited away. When a question is answered, mark it resolved and link the
  artifact that resolved it; do not delete the row.
