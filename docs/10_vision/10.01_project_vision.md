# 10.01 — Project Vision

## One-sentence pitch

Run Wallpaper Engine live wallpapers — animated shader scenes, not just
video — as the desktop background on GNOME Shell under Wayland, on
hardware where the existing Wayland path does not work.

## The problem

Wallpaper Engine is a Windows application. Its wallpapers (scenes with
shaders, video, web) are bought and subscribed through the Steam
Workshop. On Linux, [`linux-wallpaperengine`](https://github.com/Almamu/linux-wallpaperengine)
reimplements the runtime: it reads the wallpaper files the user already
owns and renders them with OpenGL — parsing `scene.json`, translating
HLSL shaders to GLSL, unpacking `.pkg` archives, playing video.

That renderer already runs on Wayland — but only on compositors that
implement the `wlr-layer-shell` protocol (Hyprland, Sway, other wlroots
compositors). It uses that protocol to place a surface in the background
layer, beneath the desktop icons.

**GNOME's compositor (Mutter) does not implement `wlr-layer-shell`, and
will not.** So on Ubuntu with GNOME on Wayland — the default desktop on
the most common Linux distribution — no external program can draw behind
the desktop. The only sanctioned way to paint into the background layer
of GNOME Shell on Wayland is a **GNOME Shell extension**: code that runs
inside the `gnome-shell` process and can insert a Clutter actor into the
background. Projects like Hanabi do exactly this — but they show only
video (via GStreamer/mpv), not Wallpaper Engine scenes.

## The insight

The runtime does not need to be rewritten — it exists and works. The gap
this project closes is the **bridge between a working renderer and GNOME
Shell on Wayland**: getting frames out of `linux-wallpaperengine` and
into a Clutter actor in the GNOME background layer, without an expensive
per-frame copy.

## Who it is for

- **Primary:** GNOME-on-Wayland users (Ubuntu first-class) who own
  Wallpaper Engine on Steam and want their scene wallpapers on the
  desktop, the way they work on Windows.
- **Secondary:** the upstream `linux-wallpaperengine` project, which
  gains a headless/PipeWire output path useful beyond GNOME (any consumer
  that can take a PipeWire video stream).

## What "done" looks like (north star)

A user on Ubuntu with GNOME on Wayland installs the extension, picks a
scene wallpaper they already own in Steam, and sees it animating as their
desktop background — paused automatically when a window is fullscreen, to
spare the GPU and battery, the way Wallpaper Engine does on Windows.

## Deliberately small first target (MVP)

Scene-type wallpapers, a single monitor, no web wallpapers, no
audio-reactivity. See `10.03_non_goals.md` for the full boundary and
`90_open_questions/90.01_open_questions.md` for what is deferred rather
than rejected.
